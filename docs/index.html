flappy_rl

教计算机玩一个自己写的 Flappy Bird，借此理解强化学习（reinforcement learning）。

这个项目的目的不是刷分，也不是手写算法，而是理解 RL 在实践中如何落地： policy 是什么、训练循环如何成形、尤其是如何设计 reward、以及如何避免 reward hacking。算法部分用现成的 Stable-Baselines3，我们把精力 放在 environment 和 reward 上——那才是真正决定成败、也最容易翻车的地方。

基本思路

游戏写成一个普通 Python 对象，用标准的 Gymnasium 接口（reset() / step()）包装，让算法直接读到小鸟的内部状态（高度、垂直速度、下一根 管道的缝隙），而不是去看屏幕、模拟按键。训练时无窗口、快于实时；每一局会被录成一个 JSON artifact，下面这个播放器就用它把那一局重演出来。

我们做两个 reward baseline 做对比：版本一是最纯粹的 sparse reward（过管 +1、死亡 −1、其余为 0），版本二在此之上加一个连续的 对齐引导（离缝隙中心越远每帧扣得越多）。版本一会让你看到「sparse 学得慢」，版本二 会让你看到「加了引导学得快、但引导本身可能被钻空子」。
