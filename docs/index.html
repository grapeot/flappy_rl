flappy_rl

教计算机玩一个自己写的 Flappy Bird，借此理解强化学习（reinforcement learning）。

这个项目的目的不是刷分，也不是手写算法，而是理解 RL 在实践中如何落地： policy 是什么、训练循环如何成形、尤其是如何设计 reward、以及如何避免 reward hacking。算法用现成的 Stable-Baselines3（PPO），精力放在 environment 和 reward 上——那才是真正决定成败、也最容易翻车的地方。

基本思路

游戏写成普通 Python 对象，用标准 Gymnasium 接口（reset() / step()）包装，让算法直接读到小鸟的内部状态（高度、垂直速度、下一根 管道缝隙的相对位置），而不是看屏幕、模拟按键。训练无窗口、快于实时；每一局录成一个 JSON，下面的播放器用它重演。

我们做两个 reward baseline：版本一是最纯粹的 sparse reward （过管 +1、死亡 −1、其余 0）；版本二在此之上加一个连续的对齐引导 （离缝隙中心越远每帧扣得越多）。下面的结果会让你看到二者的真实差异。

训练曲线：reward 和 task metric 同步上升

每张图两条线：红线是 reward（被优化的东西）、 青线是过管数 score（我们真正想要的 task metric）。 两者同向上升说明没有 reward hacking——若哪天 reward 涨而 score 不涨， 那条背离就是作弊的信号。两版对比能看出：sparse 学得更晚、更抖；shaped 学得更早、更稳。

泛化测试：在没训练过的 seed 上评估

RL 的常见陷阱是 policy 只记住了训练时的特定关卡。我们在 30 个全新 seed （训练时从没见过）上评估，看它是不是真学会了玩。结果两版几乎都完美——说明相对坐标 observation 让 policy 学到了通用策略，而非死记硬背。

版本平均过管数中位最低打满率

有意思的细节：sparse 版（100%）在泛化上反而略胜 shaped（96.7%）。因为 sparse 优化的 就是纯粹的"过管"，没有对齐引导来分散；shaped 学得快，但那个对齐惩罚是个代理目标， 会把策略轻微地往"对齐"而非"纯过管"上拉——这正是 shaped reward 那个根本张力的温和体现。
